# tasks/ — Workflow Workspace

## Purpose
Operational tracking for the current project cycle. Active task lists and self-improvement log.

## File Inventory
| File | Description |
|------|-------------|
| todo.md | Active task tracker — checkable items for current work session |
| lessons.md | Self-improvement log — patterns from corrections, rules to prevent recurrence |

## Conventions
- **todo.md**: Use `- [ ]` / `- [x]` checkable items. Add review section after completion.
- **lessons.md**: Log pattern, not just the fix. Include: what happened, why, rule to prevent it.
- **State via naming**: If tasks need versioning, use `draft-v1.md`, `plan-v2.md` etc.

## When to Read
- At session start — review `lessons.md` for relevant patterns before beginning work
- During execution — check `todo.md` for current progress

## When to Update
- **todo.md**: Before starting work (plan), during work (track), after work (document)
- **lessons.md**: Immediately after ANY correction from Yousef
