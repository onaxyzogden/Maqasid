# Voice & Tone — Maqasid OS V2.1

## Islamic Terminology Standards

### Transliteration Convention
Use simplified Arabic transliteration without diacritical marks in code and UI labels. Use full diacritics only in Quranic citations and hadith references.

| Arabic | Transliteration | Usage Context |
|--------|----------------|---------------|
| أمانة | Amanah | Trust/stewardship — used in Amanah Gate, BBOS qualification |
| صدق | Sidq | Truthfulness — self-accounting standard |
| إتقان | Itqan | Excellence in craft — quality standard |
| توكل | Tawakkul | Reliance on Allah — post-effort trust |
| بركة | Barakah | Divine blessing — BBOS brand core |
| مقاصد | Maqasid | Higher objectives — app name, pillar system |
| وقف | Waqf | Endowment — generosity framework |
| صدقة | Sadaqah | Charity — generosity framework |
| غرر | Gharar | Deception/uncertainty — prohibited in transactions |
| ربا | Riba | Usury/interest — prohibited |

### Seven Maqasid (Pillar Names)
1. **Faith** (حفظ الدين — Hifz al-Din)
2. **Life** (حفظ النفس — Hifz al-Nafs)
3. **Intellect** (حفظ العقل — Hifz al-Aql)
4. **Family** (حفظ النسل — Hifz al-Nasl)
5. **Wealth** (حفظ المال — Hifz al-Mal)
6. **Environment** (حفظ البيئة — Hifz al-Bi'ah)
7. **Ummah** (حفظ الأمة — Hifz al-Ummah)

## UI Copy Voice

### Tone
- Purposeful, not preachy — let the structure embody the values
- Professional with warmth — this is a business OS, not a lecture
- Direct and actionable — avoid passive voice and filler
- Respectful of Islamic framing — never genericize or strip religious context

### Patterns
- **Button labels**: Action-first, concise ("Add Expense", "Start Ceremony", "Submit Gate")
- **Empty states**: Encouraging, forward-looking ("No tasks yet — bismillah, let's begin")
- **Error messages**: Honest, non-blaming ("Could not save — check your connection and try again")
- **Confirmations**: Clear consequence ("This will permanently remove the contact. Continue?")
- **Section headers**: Noun-based, not verb-based ("Prayer Times", not "View Prayer Times")

### What to Avoid
- Generic corporate language that strips Islamic identity
- Over-explaining — trust the user's intelligence
- Passive voice in action contexts
- Emojis in formal UI elements (acceptable in informal notes/journals)
