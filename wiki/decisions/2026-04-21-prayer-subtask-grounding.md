# Decision: Full structured-source migration of the 18 prayer-slide-up boards

## Context

The two-axis grounding schema (`provenanceTier` × `relevance`) was formalised on 2026-04-18
([2026-04-18-milos-grounding-two-axis.md](./2026-04-18-milos-grounding-two-axis.md)) with a
lint harness (`scripts/lint-grounding.mjs`), but zero pillars had migrated. At session start,
`lint-grounding` reported 1,936 legacy free-form `sources` strings across the codebase and
0 structured `GroundingSource[]` arrays.

The prayer slide-up ([2026-04-21-prayer-slide-up-fln.md](./2026-04-21-prayer-slide-up-fln.md))
routes tasks into 18 first-class boards `prayer_{fajr|dhuhr|asr|maghrib|isha|tahajjud}_{before|during|after}` —
the highest-signal, most-cited surface in the app — making it the natural pilot for the
structured-source rollout.

## Decision

Migrate every subtask reachable from the 18 `prayer_*` boards to the structured
`GroundingSource[]` shape defined in `docs/grounding-schema.md` §3. Keep legacy-string
rendering untouched for unrelated pillars — the array branch is opt-in via
`Array.isArray(subtask.sources)`.

### Schema applied per subtask

```js
{
  title, done, tier, amanahRationale,          // existing
  why:  "≤3 sentences — theological framing",  // NEW
  how:  "≤3 sentences — operational steps",    // NEW
  sources: [ GroundingSource, ... ],           // NEW array shape
}
```

Each `GroundingSource` carries `{ kind, ref, arabic, translation, relevance,
provenanceTier, hadithGrade?, rationale }`. **Arabic and translation are always populated
directly on the entry** — bypassing the fawazahmed0-CDN-generated `hadith.js` bundle so
cards do not depend on bundle completeness.

### Dual-corpus authoring pattern

- **Muslim Scholar NotebookLM** (`1c17b03b-3537-4fde-b5ba-562dbe0c1aab`) — theological
  framing: `why`, per-source `rationale`. Verified source of truth for Arabic and translation.
- **Direct Quran/Hadith MCP** — canonical text normalisation when needed (`fetch_quran`,
  `fetch_hadith`, `fetch_translation`).
- **How?** is operational, not grounded — authored from task semantics.

### Tier-label rename

`amanah-tiers.js` T3 display label renamed **"Aspiration" → "Niyyah"** to align with
the canonical [Amanah Gate Protocol](../concepts/amanah-gate-protocol.md) universal
tier names.

### Transliteration rule (English-field gap standard)

Any transliterated Arabic phrase rendered in an English field **must** be followed by
its translation in parentheses. Applies both to structured `translation` fields and to
`src/data/hadith-overrides.js` entries (the shallow-merge layer over the auto-generated
`hadith.js`).

## Consequences

### Migrated surface

- **18 prayer boards** — `fajr|dhuhr|asr|maghrib|isha|tahajjud` × `before|during|after`.
- **87 subtasks** — 100% grounded (`total: 87, grounded: 87, legacy: 0`).
- **PRAYER_GUIDE structure rows** in `prayer-seed-tasks.js` extended with full
  `tier/amanahRationale/why/how/sources` fields; builder `buildPrayerSeedTasks()` passes
  them straight through into subtask shape.
- **FAITH_SEED_TASKS** — 7 parent tasks / 20 distinct subtasks migrated from free-form
  Markdown `sources` strings to structured arrays. Generic sunan edited once at canonical
  source; duplication across the five daily prayers propagates automatically.

### Rendering

- `TaskDetailPanel` subtaskDetailView: three labelled sections (Why / How / Sources) when
  array shape is present; fallback to legacy Markdown `description` otherwise.
- `SubtaskSources`: array-branch renders each entry via `QuranEmbed` or `HadithCard` with
  `<AmanahTierBadge tier={provenanceTier}/>` + `<RelevanceChip value={relevance}/>` +
  existing hadith-grade chip.
- `RelevanceChip` component + `relevance-chips.js` config added.
- Grounded pill at subtask detail top (green when ≥1 grounded entry, amber otherwise).

### Bundle-gap standard procedure

For any `HadithCard` that renders from the bundle and shows a transliterated-but-untranslated
English: add a keyed entry to `src/data/hadith-overrides.js` with only the fields being
corrected. **Never edit `src/data/hadith.js` directly** — it is auto-generated by
`scripts/fetch-hadith.py` and will be overwritten. Structured-array `sources[]` entries
render from their own `arabic`/`translation` and do not consult the bundle.

### Verification

- `npm run build` clean (1.68s, 2703 modules).
- `node -e "import('./src/data/seed-tasks/prayer-seed-tasks.js')..."` confirms
  18 boards / 87 subtasks / 0 legacy.
- `npm run lint` — 625 pre-existing unrelated errors (no-unused-vars, no-empty,
  `__dirname` in vite.config.js). None introduced by this session.
